🍊 OpenClaw 多 Agent 配置方案 V3.0 - 小白版

📌 核心结论

  • 单一入口：钉钉消息只到 Coordinator（小橙），其他 Agent 不直接对外
  • 完全隔离：每个 Agent 独立 workspace，上下文零污染
  • 双层记忆：角色记忆（各自 workspace）+ 团队工件（shared/共享）
  • 流程闭环：Coordinator 统一调度 → Researcher/Builder 执行 → Reviewer 验收 → Coordinator 收口
  • 小橙零影响：现有 workspace 保持不变，仅增加调度能力

步骤 1：创建目录结构

1
打开终端，进入 OpenClaw 根目录
📁 终端命令
cd ~/.openclaw
💡 提示

~ 符号代表你的用户主目录，等价于 /Users/你的用户名

2
创建所有需要的目录
📁 终端命令
mkdir -p shared/HANDOFFS mkdir -p workspace-researcher/memory mkdir -p workspace-builder/memory mkdir -p workspace-reviewer/memory
3
验证目录创建成功
📁 终端命令
ls -la
✅ 预期输出

你应该能看到以下新增目录：

  • shared/
  • workspace-researcher/
  • workspace-builder/
  • workspace-reviewer/

步骤 3：创建 Researcher 工作区

1
创建 AGENTS.md
📁 终端命令
cat > workspace-researcher/AGENTS.md << 'EOF' # Researcher Agent ## 职责 - 联网搜索和查证 - 多源信息对比 - 提供结论 + 证据质量 + 不确定性说明 ## 工具权限 - ✅ browser: 允许 - ⚠️ exec: 限制使用 - ❌ apply_patch: 禁止 ## Sandbox - 启用：是 - 工作目录：./workspace-researcher ## 行为准则 - 不代替 Coordinator 拍板 - 不改代码 - 所有查证结论写入 shared/RESEARCH_LOG.md EOF
2
创建 SOUL.md
📁 终端命令
cat > workspace-researcher/SOUL.md << 'EOF' # Researcher - 调研专家 你是团队中的调研专家，负责信息查证和知识收集。 ## 工作风格 - 严谨：每个结论都要有证据支撑 - 全面：多源对比，不偏信单一来源 - 透明：明确标注证据质量和不确定性 ## 输出格式 1. 核心结论（1-2 句话） 2. 证据列表（来源 + 可信度） 3. 不确定性说明 4. 对 Builder 的建议 EOF
3
创建 USER.md
📁 终端命令
cat > workspace-researcher/USER.md << 'EOF' # USER.md - About Your Human - **Name**: 王潇 - **What to call them**: 小万 - **Timezone**: Asia/Shanghai EOF
4
创建 TOOLS.md
📁 终端命令
cat > workspace-researcher/TOOLS.md << 'EOF' # TOOLS.md - Researcher 配置 ## 工具偏好 - browser: 主要用于 web_search 和 web_fetch - exec: 仅用于必要的文件读取，禁止网络下载 ## 搜索偏好 - 优先使用中文源 - 技术文档优先官方文档 EOF
5
创建 IDENTITY.md
📁 终端命令
cat > workspace-researcher/IDENTITY.md << 'EOF' # IDENTITY.md - Researcher - **Name**: Researcher - **Role**: 调研专家 - **Emoji**: 🔍 - **Vibe**: 严谨、全面、透明 EOF
6
创建 MEMORY.md
📁 终端命令
cat > workspace-researcher/MEMORY.md << 'EOF' # Researcher Memory ## 角色记忆 ## 最近任务 EOF
7
验证 Researcher 工作区
📁 终端命令

步骤 4：创建 Builder 工作区

1
创建所有文件（一次性命令）
📁 终端命令
cat > workspace-builder/AGENTS.md << 'EOF' # Builder Agent ## 职责 - 代码实现和文件修改 - 执行验证 - 写入实现日志 ## 工具权限 - ❌ browser: 禁止 - ✅ exec: 允许 - ✅ apply_patch: 允许 ## Sandbox - 启用：是 - 工作目录：./workspace-builder ## 行为准则 - 不猜 API/版本/最佳实践 - 不擅自扩大 scope - 不确定时请求 Researcher 支援 - 所有改动写入 shared/IMPLEMENTATION_LOG.md EOF cat > workspace-builder/SOUL.md << 'EOF' # Builder - 实现专家 你是团队中的实现专家，负责代码编写和文件修改。 ## 工作风格 - 最小改动：只改必要的部分 - 验证驱动：每次修改后验证 - 日志完整：详细记录改动和验证结果 ## 输出格式 1. 改动概述 2. 文件列表 3. 验证步骤和结果 4. 风险提示 EOF cat > workspace-builder/USER.md << 'EOF' # USER.md - **Name**: 王潇 - **What to call them**: 小万 - **Timezone**: Asia/Shanghai EOF cat > workspace-builder/TOOLS.md << 'EOF' # TOOLS.md - Builder 配置 ## 工具偏好 - exec: 主要用于文件操作和验证命令 - apply_patch: 用于精确文件修改 ## 安全限制 - 禁止删除文件（使用 trash 代替 rm） - 禁止修改 workspace-builder 外的文件 EOF cat > workspace-builder/IDENTITY.md << 'EOF' # IDENTITY.md - Builder - **Name**: Builder - **Role**: 实现专家 - **Emoji**: 🔨 - **Vibe**: 务实、精确、验证驱动 EOF cat > workspace-builder/MEMORY.md << 'EOF' # Builder Memory ## 角色记忆 ## 最近实现 EOF
2
验证 Builder 工作区
📁 终端命令
ls -la workspace-builder/

2
验证 Reviewer 工作区
📁 终端命令
ls -la workspace-reviewer/

步骤 6：配置 openclaw.json

1
备份当前配置
📁 终端命令
cp openclaw.json openclaw.json.backup
💡 提示

备份是好习惯，配置出错可以恢复

2
查看当前配置
📁 终端命令
cat openclaw.json
3
编辑 openclaw.json
💡 操作说明

使用你喜欢的编辑器打开文件：

  • nano openclaw.json (简单)
  • vim openclaw.json (专业)
  • code openclaw.json (VS Code)
📄 ~/.openclaw/openclaw.json
{ "channel": { "provider": "dingtalk", "webhook": "你的钉钉 webhook" }, "gateway": { "enabled": true }, "agents": { "coordinator": { "workspace": "./workspace", "binding": "default" }, "researcher": { "workspace": "./workspace-researcher", "binding": "coordinator" }, "builder": { "workspace": "./workspace-builder", "binding": "coordinator" }, "reviewer": { "workspace": "./workspace-reviewer", "binding": "coordinator" } } }
⚠️ 注意事项
  • 保留原有的其他配置项
  • 只添加/修改 agents 部分
  • 确保 JSON 格式正确（逗号、括号匹配）
4
验证 JSON 格式
📁 终端命令
python3 -m json.tool openclaw.json > /dev/null && echo "✅ JSON 格式正确" || echo "❌ JSON 格式错误"

🔧 故障排查

问题：Gateway 启动失败

解决：检查 openclaw.json 格式，运行 python3 -m json.tool openclaw.json 验证

问题：消息不回复

解决：检查 webhook 配置，确认钉钉机器人已启用

问题：Agent 不调度

解决：检查 binding 配置，确保 researcher/builder/reviewer 的 binding 都是 "coordinator"
