🍊 OpenClaw 多 Agent 配置方案 V3.0

📌 核心结论

  • 单一入口：钉钉消息只到 Coordinator（小橙），其他 Agent 不直接对外
  • 完全隔离：每个 Agent 独立 workspace，上下文零污染
  • 双层记忆：角色记忆（各自 workspace）+ 团队工件（shared/共享）
  • 流程闭环：Coordinator 统一调度 → Researcher/Builder 执行 → Reviewer 验收 → Coordinator 收口
  • 小橙零影响：现有 workspace 保持不变，仅增加调度能力

一、架构设计

~/.openclaw/
├── openclaw.json # 主配置（共享 gateway/channel）
├── shared/ # 团队共享工件
│ ├── TASK_BOARD.md
│ ├── DECISIONS.md
│ ├── RESEARCH_LOG.md
│ ├── IMPLEMENTATION_LOG.md
│ ├── REVIEW_LOG.md
│ └── HANDOFFS/
├── workspace/ # 小橙 (Coordinator) 主工作区【保持不变】
│ └── AGENTS.md, SOUL.md, USER.md, TOOLS.md, MEMORY.md, memory/
├── workspace-researcher/ # Researcher 独立工作区
│ └── AGENTS.md, SOUL.md, USER.md, TOOLS.md, MEMORY.md, memory/
├── workspace-builder/ # Builder 独立工作区
│ └── AGENTS.md, SOUL.md, USER.md, TOOLS.md, MEMORY.md, memory/
└── workspace-reviewer/ # Reviewer 独立工作区
└── AGENTS.md, SOUL.md, USER.md, TOOLS.md, MEMORY.md, memory/

二、核心配置

2.1 openclaw.json 配置

{ "channel": { "provider": "dingtalk", "webhook": "你的钉钉 webhook" }, "gateway": { "enabled": true }, "agents": { "coordinator": { "workspace": "./workspace", "binding": "default" }, "researcher": { "workspace": "./workspace-researcher", "binding": "coordinator" }, "builder": { "workspace": "./workspace-builder", "binding": "coordinator" }, "reviewer": { "workspace": "./workspace-reviewer", "binding": "coordinator" } } }

2.2 工具权限与 Sandbox 策略

Agent Sandbox exec browser 职责边界
Coordinator off ✅ ✅ 调度、汇总、不深度实现
Researcher on ⚠️ ✅ 查证、不拍板、不改代码
Builder on ✅ ❌ 实现、不猜 API、不扩 scope
Reviewer on ❌ ❌ 验收、不重做实现

三、协作流程

用户 → Coordinator (小橙)
↓
Researcher
(信息不足时)
Builder
(目标清晰时)
↓
Reviewer
↓
Coordinator → 用户

标准闭环

  1. Coordinator 接需求 → 拆解任务
  2. 信息不足 → 派 Researcher 查证
  3. 目标清晰 → 派 Builder 实现
  4. 完成后 → 交 Reviewer 验收
  5. Reviewer 返回结论 → Coordinator 决定：
    • ✅ Accept（完成）
    • 🔄 Revise（修改）
    • ⚠️ Escalate（升级/补研/人工决策）

四、双层记忆设计

层级 位置 内容 访问范围
角色记忆 各 workspace/MEMORY.md 常用偏好、角色经验、任务习惯、稳定规则 仅本 Agent
团队工件 shared/ 目录 当前目标、状态、决策、查证结论、实施改动、审查结果 所有 Agent 可读

五、共享工件模板

5.1 TASK_BOARD.md

# Goal # Constraints # Backlog # In Progress # Review # Blocked # Done # Next action # Owner

5.2 DECISIONS.md

# Decisions ## YYYY-MM-DD - decision - why - impact

5.3 HANDOFFS/README.md

# Handoffs Each handoff should record: - from / to - task / current state - blockers / expected output - related files

六、小橙 (Coordinator) 保障措施

✅ 你的使用状态完全不受影响：
  • 入口唯一：所有钉钉消息只到你这里，其他 Agent 不直接对外
  • 工作区独立：你的 ./workspace 保持原样，不与其他 Agent 混用
  • 调度权：你决定何时调用哪个 Agent，调用顺序由你控制
  • 收口权：所有结果汇总到你这里，由你统一回复用户
  • 记忆独立：你的 MEMORY.md 只记录你的决策和上下文，不被污染

七、安全规则

⚠️ 10 轮失败挂起规则：

同一任务 unresolved 超过 10 轮 → 暂停、记录 attempts/errors/blockers、上报人工决策，禁止无限循环试错。
